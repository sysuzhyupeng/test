5、fetch请求跨域：添加：mode: 'no- cors'
设置了no-cors以后就不会发送跨域请求了,所以返回 status=0,因为请求根本没发出去.
正确的做法是
https://developer.mozilla.org/zh-TW/docs/Web/HTTP/Access_control_CORS#Access-Control-Allow-Origin
让服务器开发人员在Http Response 中添加Access-Control-Allow-Origin 头

6、移除scroll事件的坑     http://react-china.org/t/react-scroll/7192
问题：设置了 window.addEventListener('scroll', this.handleScroll);     componentWillUnmount时要移除事件 window.removeEventListener('scroll', this.handleScroll);
描述：这样虽然可以，但是在handleScroll 事件里面 this不是指向当前组件；
          或者设置window.addEventListener('scroll', this.handleScroll.bind(this)); 和window.removeEventListener('scroll', this.handleScroll.bind(this)); 这样无效，因为两个bind(this)分别指向不同的匿名方法，所以无法移除事件
解决方法：应该在constructor设置 this.handleScroll = this.handleScroll.bind(this);

11、Warning: Accessing PropTypes via the main React package is deprecated. Use the prop-types package from npm instead.
https://facebook.github.io/react/blog/2017/04/07/react-v15.5.0.html
原因：在react 15.5 版本之后，PropTypes不从react引入，而是通过NPM安装prop-types包获取
// Before (15.4 and below)import React from 'react';

class Component extends React.Component {
  render() {
    return <div>{this.props.text}</div>;
  }}

Component.propTypes = {  text: React.PropTypes.string.isRequired,}

// After (15.5)import React from 'react';import PropTypes from 'prop-types';class Component extends React.Component {
  render() {
    return <div>{this.props.text}</div>;
  }}

Component.propTypes = {  text: PropTypes.string.isRequired,};

16、react获取url查询参数：this.props.location.query
链接跳转传递参数：<Link to={{pathname:"filter", query:{name: 'pig', id: 55}, state:{a: 1, b: 2}}} >aaaaaaaaaa</Link>
跳转过去的链接为：http://c.dev.8891.com.tw:8080/m/filter?id=55&name=pig
this.props.location.query 得到：{id: "55", name: "pig"}
this.props.location.state 得到：{a: 1, b: 2}
但是刷新页面后this.props.location.query还可以拿到，this.props.location.state而this.props.location.state拿不到了
